## Anti-Patterns Verdict

**LLM assessment:** No, not on the surface. Signal Room aesthetic is committed and specific — atmospheric gradient, amber-only stat readouts, indigo channel locks. No gradient text, no glassmorphism, no eyebrows on every section. Second-order check passes: the execution is specific enough to hold against "dark portfolio with indigo filter chips" territory. One genuine slop tell: 51-tag filter bar. An editorial hand would have curated to 8 macro-categories. Raw technology name explosion ("yt-dlp", "Whisper", "MarkItDown" as peer-level filters alongside "AI" and "Next.js") reads as unedited output.

**Deterministic scan:** detect.mjs returned zero violations across all 15 source files. Clean after the full impeccable pass.

## Overall Impression

The site has genuine personality. The Signal Room holds as an aesthetic identity, the typography is doing real work at the hero, and the contact form is more robust than 95% of portfolio sites. The one place where the design stops being intentional is the project filter — 51 visible chips at the same visual weight is a cognitive Wall of Options that undermines the clean-data-room vibe.

Score movement: 28 → 32. The remaining 8 points are genuine gaps.

## What's Working

1. **Signal Room identity is committed.** Two-accent constraint, amber lock, flat-by-default cards — every element earns its place. This level of design constraint at a portfolio scale is rare.
2. **Hero post-bolder pass.** Space Grotesk 700 at text-5xl/7xl with "— fast." in indigo-300. Direct, legible, doesn't need to shout.
3. **Contact form depth.** Server-side validation, client-side constraints, preserved state on error, role="alert", fallback email path. Genuinely hardened.

## Priority Issues

**[P1] 51 filter tags is a cognitive Wall of Options**
- Why it matters: Miller's Law revised — 51 atomic tech names simultaneously (yt-dlp, Whisper, MarkItDown, AI, Next.js...) guarantees skip or abandonment. Single biggest UX liability on homepage.
- Fix: Replace with 6-8 curated macro-categories: All / AI & LLM / Dashboards & Data / Automation / Next.js / Python / Full-Stack / Infrastructure. Map project tags to categories in data layer. 51 granular tags stay on cards as metadata.
- Suggested command: /impeccable distill

**[P1] CaseStudiesList token drift: border-white/10 / bg-white/5 hardcoded**
- Why it matters: Uses border-white/10 and bg-white/5 directly (lines 27, 52) instead of border-border and bg-card CSS variables. Design system token updates won't propagate here. Maintenance liability.
- Fix: Replace border-white/10 → border-border, bg-white/5 on tags → bg-muted or bg-card equivalent. Audit full file including CV page.
- Suggested command: /impeccable harden

**[P2] Contact page eyebrow contradicts DESIGN.md anti-pattern rule**
- Why it matters: app/contact/page.tsx opens with "Get in touch" in text-xs font-mono uppercase above the h1. DESIGN.md explicitly bans this pattern: "The hero has one location callout; that is the only standalone eyebrow on the page."
- Fix: Remove the "Get in touch" eyebrow. The h1 and page URL provide sufficient context.
- Suggested command: /impeccable quieter

**[P2] Nav has no current-page indicator**
- Why it matters: Five nav items at equal visual weight on all pages. No aria-current, no active state, no visual location signal. Heuristic 1 gap.
- Fix: Add aria-current="page" to matching nav link. Apply visual active state (text-foreground vs text-foreground/60, or subtle indigo underline).
- Suggested command: /impeccable harden

**[P3] text-wrap: pretty missing on About section prose**
- Why it matters: max-w-prose is set but no text-wrap: pretty — orphaned words at some viewport widths.
- Fix: Add [text-wrap:pretty] to prose div in about.tsx.
- Suggested command: /impeccable typeset
