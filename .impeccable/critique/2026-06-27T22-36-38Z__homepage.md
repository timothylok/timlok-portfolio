## Anti-Patterns Verdict

**LLM assessment:** Does not read as AI-generated. Clay/terracotta palette, Bricolage Grotesque, and "Builder's Notebook" concept are coherent and distinctive. Dark-SaaS anti-reference successfully avoided.

The tell creeping in is the header decoration template: 1px vertical left clay line + radial gradient + `h-px w-36 bg-gradient-to-r from-clay-500/50 to-transparent` gradient underline appearing on CV, CaseStudies index, and CaseStudy detail in near-identical form. Three pages with the same header treatment reads as stamped.

**Deterministic scan:** 5 findings, all false positives: console-egg.tsx colors (easter egg palette, not production), Georgia and Courier New in print block (intentional). Zero real design system violations.

## Overall Impression

Genuine identity, disciplined code, cohesive clay system. Three gaps: broken contact form CTA (white button on white card), probable hero h1 overflow at 390px, and template-pattern header spreading to inner pages. Fix those and the site feels done.

## What's Working

1. Clay design system is genuinely tonal — token discipline strong, no bg-white/5 violations, clay only on interactive surfaces.
2. Filter chips are mobile-first done right — horizontal scroll bleed pattern, AnimatePresence layout animations.
3. Contact form logic is complete — required fields, email type, dropdown constraints, error/success states.

## Priority Issues

**[P1] Hero h1 overflows at 390px mobile**
- text-5xl (48px) with hardcoded <br /> means "Building AI-powered" is a single ~480px line in a 342px container.
- Fix: Use `clamp(2.5rem, 8vw, 4.5rem)` instead of text-5xl sm:text-7xl. Remove hardcoded <br />, let text-wrap:balance break naturally.
- Suggested command: /impeccable adapt

**[P1] Contact form submit button invisible on light theme**
- contact-form.tsx:138 — bg-white text-black hover:bg-white/90 on a white card. No border. Invisible.
- Fix: Change to bg-clay-500 hover:bg-clay-600 text-white with clay glow shadow, matching hero/CTA buttons.

**[P1] CaseStudy detail page has banned eyebrow label**
- CaseStudies/[slug]/page.tsx:56 — "Case Study" in text-xs font-mono uppercase tracking-widest above h1. Violates explicit rule.
- Fix: Delete that <p> element entirely.

**[P2] Inner page header pattern becoming template**
- CV, CaseStudies index, CaseStudy detail share: 1px clay vertical line + radial gradient + gradient underline. 4 gradient underlines across the site.
- Fix: Limit gradient underline to hero. Differentiate or simplify inner page headers.
- Suggested command: /impeccable distill

**[P2] Mobile nav drawer has no animation**
- nav.tsx:65 — instant show/hide. Inconsistent with rest of site's motion language.
- Fix: AnimatePresence with opacity+y slide, 200ms easeOut, reduced-motion safe.
- Suggested command: /impeccable animate
